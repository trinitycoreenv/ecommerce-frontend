Key Relationships

User ↔ Vendor (1:1)

Each user can have one vendor profile. Vendors are users with additional business information and subscription details.

Vendor ↔ Subscription (1:Many)

A vendor can have multiple subscription records over time, tracking their subscription history and changes.

Vendor ↔ Product (1:Many)

Each vendor can create and manage multiple products in their catalog.

Category ↔ Product (1:Many)

Products are organized into categories with specific policy rules and approval workflows.

User ↔ Order (1:Many)

Customers can place multiple orders, while vendors can receive orders for their products.

Order ↔ OrderItem (1:Many)

Each order contains multiple items, linking to specific products with quantities and prices.

Order ↔ Shipment (1:Many)

Orders can have multiple shipments for different items or split deliveries.

Vendor ↔ LogisticsConfig (1:1)

Each vendor has specific logistics configuration for shipping zones, carriers, and SLA definitions.

Order ↔ Transaction (1:Many)

Orders generate transactions for payment processing, commission calculations, and payout tracking.

Vendor ↔ Payout (1:Many)

Vendors receive multiple payouts based on their transaction history and commission structure.

User ↔ AuditLog (1:Many)

All user actions are logged for security, compliance, and audit trail purposes.

Report Generation

Reports are generated by users and contain aggregated data from multiple entities for business intelligence.
